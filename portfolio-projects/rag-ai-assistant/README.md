# 🛠️ ИИ RAG-Бот с базой знаний / AI RAG-Assistant Bot

Асинхронный Telegram-бот на базе **aiogram 3.x** и **API Google Gemini**, реализующий архитектуру **RAG (Retrieval-Augmented Generation)**. 
Позволяет загружать личные текстовые документы (`.txt`), автоматически индексирует их в векторную базу данных и отвечает на вопросы пользователей, используя только контекст загруженных документов.

---

## 🌟 Ключевые особенности / Features

- **Асинхронность**: Полный неблокирующий стек на `aiogram 3.x` с выносом тяжелых API-вызовов в пул потоков через `asyncio.to_thread`.
- **Локальный Vector Store**: Легковесное векторное хранилище на базе `NumPy` с использованием косинусного сходства (Cosine Similarity). Никаких тяжелых бинарных C++ библиотек (вроде FAISS), которые ломают установку на Windows.
- **Интеграция ИИ**: Использование модели `models/text-embedding-004` для генерации векторных представлений текста и `gemini-1.5-flash` для генерации ответов.
- **Динамическое индексирование**: Загрузка документов в реальном времени через отправку файла `.txt` боту.
- **Встроенная база знаний**: Если пользователь не загрузил свои файлы, бот автоматически отвечает по встроенной базе знаний о студии **Nexus Labs** (`knowledge.txt`).

## 🚀 Установка и запуск / Installation & Setup

### 1. Перейдите в папку проекта:
```bash
cd portfolio-projects/rag-ai-assistant/
```

### 2. Установите зависимости:
```bash
pip install -r requirements.txt
```

### 3. Настройте конфигурацию:
Создайте файл `.env` из шаблона `.env.example`:
```bash
cp .env.example .env
```
Заполните переменные окружения своими ключами:
- `BOT_TOKEN` — токен вашего Telegram-бота (можно получить у [@BotFather](https://t.me/BotFather)).
- `GEMINI_API_KEY` — API-ключ Google Gemini (можно получить в [Google AI Studio](https://aistudio.google.com/)).

### 4. Запустите бота:
```bash
python bot.py
```

## 💡 Как это работает / Architecture

1. **Загрузка**: При отправке `.txt` файла бот считывает текст и передает его в `DocumentProcessor`.
2. **Чанкинг (Chunking)**: Текст разбивается на части (по умолчанию 500 символов с перекрытием 100 символов) с сохранением целостности предложений.
3. **Эмбеддинги (Embeddings)**: Для каждого чанка генерируется 768-мерный вектор через модель `text-embedding-004`.
4. **Поиск (Retrieval)**: Когда пользователь задает вопрос, его текст также переводится в вектор, и с помощью косинусного сходства в `LocalVectorStore` находятся топ-3 наиболее близких по смыслу текстовых чанка.
5. **Генерация (Generation)**: Найденные чанки добавляются в промпт модели `gemini-1.5-flash` в качестве контекста, гарантируя точный и обоснованный ответ без выдумок (галлюцинаций).
