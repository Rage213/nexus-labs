# ⚡ Асинхронный WebSocket Монитор Цен / Real-Time WebSocket Price Monitor

![Интерфейс монитора](mockup.png)

Высокопроизводительный асинхронный скрипт на Python для отслеживания сделок и колебаний цен криптовалютных пар в реальном времени через публичный **WebSocket API биржи Bybit (Spot V5)**. 
При фиксации резких колебаний цены (превышающих заданный лимит), скрипт мгновенно отправляет оповещение в Telegram.

---

## 🌟 Ключевые особенности / Features

- **Мгновенная реакция**: Использование постоянного WebSocket соединения вместо периодических HTTP-запросов (Polling). Цена обновляется с задержкой в миллисекунды.
- **Асинхронность**: Полностью асинхронный стек на `asyncio` и `websockets`. Сетевые операции не блокируют получение новых пакетов данных.
- **Отказоустойчивость**: Автоматическое переподключение (Auto-reconnect) с интервалом ожидания при обрыве сети или падении серверов API.
- **Неблокирующие уведомления**: Отправка сообщений в Telegram запускается в фоновом режиме через `asyncio.create_task`, гарантируя, что задержки Telegram API не вызовут лагов в обработке котировок.
- **Быстрый старт**: Монитор Bybit работает на публичных эндпоинтах и не требует создания API-ключей биржи для подписки на сделки.

## 🚀 Установка и запуск / Setup & Run

### 1. Перейдите в папку проекта:
```bash
cd portfolio-projects/websocket-price-monitor/
```

### 2. Установите зависимости:
```bash
pip install -r requirements.txt
```

### 3. Настройте конфигурацию:
Создайте файл `.env` из шаблона `.env.example`:
```bash
cp .env.example .env
```
Заполните переменные окружения:
- `TELEGRAM_BOT_TOKEN` — токен вашего бота.
- `TELEGRAM_CHAT_ID` — ID чата, куда слать оповещения.
- `PRICE_CHANGE_THRESHOLD_PERCENT` — порог изменения цены в процентах (например, `0.05` для реагирования на изменения в 0.05% от цены).

### 4. Запустите мониторинг:
```bash
# Мониторинг BTCUSDT по умолчанию
python main.py

# Или укажите любую другую спотовую пару Bybit (например, ETHUSDT)
python main.py ETHUSDT
```

## 🧬 Как это устроено / How it works

1. **Подписка**: Скрипт подключается к `wss://stream.bybit.com/v5/public/spot` и отправляет фрейм `"op": "subscribe"` с топиком `"publicTrade.BTCUSDT"`.
2. **Обработка**: При поступлении сделки извлекается текущая цена ордера.
3. **Расчет движения**: Вычисляется процентное отклонение от опорной цены (Reference Price).
4. **Оповещение**: Если отклонение превышает порог `PRICE_CHANGE_THRESHOLD_PERCENT` (например, курс вырос или упал), бот шлет уведомление, а текущая цена становится новой опорной точкой.
